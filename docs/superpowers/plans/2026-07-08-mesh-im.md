# Mesh IM Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把现有 Pager 改造成 Mesh——真人-人 IM：用户名+密码注册登录、单向加好友、1:1+群聊、按成员实时扇出+APNs 推送。

**Architecture:** 三种 Durable Object：DirectoryDO（全局单例，账号注册表+session token+搜人）、UserDO（每用户一个私有空间，好友/会话索引/设备/WS 扇出）、ConversationDO（每会话，成员名单+消息流，按成员扇出到各 UserDO）。砍掉 daemon/@AI/机器。

**Tech Stack:** Cloudflare Workers + Durable Objects (SQLite)、zod（@pager/protocol）、Vitest + `@cloudflare/vitest-pool-workers`、iOS SwiftUI (Swift 6, iOS 18, XcodeGen)、fastlane + GitHub Actions。

## Global Constraints

- 语言：代码注释/文案中文；标识符英文。用户偏好中文沟通。
- 密码：只存 PBKDF2-SHA256 hash（Web Crypto，salt 16B，迭代 ≥100000），格式 `pbkdf2$<iter>$<b64 salt>$<b64 key>`；常数时间比较。永不存明文。
- session token 形如 `stk_<32 hex>`；userId 形如 `usr_<uuid>`。
- 用户名归一化：小写，`^[a-z0-9_]{3,20}$`，全局唯一。
- 消息 author 由服务端按 session 身份盖，客户端自报不可信。
- iOS bundle id 全新 `com.wangjianshuo.Mesh`，显示名 `Mesh`，Team `97XBW2A43H`。
- 确定性 1:1 convId：两 userId 字典序排序拼 `dm_<a>_<b>`。
- 不迁移旧 Pager 数据；旧 DO 类（MachineDO）保留文件但从路由/绑定摘除。

**Interfaces (Produces):**

```ts
  expect(h.startsWith("pbkdf2$")).toBe(true);
  expect(await verifyPassword("hunter2", h)).toBe(true);
  expect(await verifyPassword("wrong", h)).toBe(false);
});
```
- [ ] **Step 2: 运行确认失败** — `cd hub && npx vitest run test/crypto.test.ts`。

- [ ] **Step 4: 运行确认通过**。
